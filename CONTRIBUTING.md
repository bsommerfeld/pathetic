# Contributing to Pathetic

Thank you for your interest in contributing to `pathetic`! This high-performance, thread-safe pathfinding library
thrives thanks to community contributions. Whether you're fixing bugs, adding features, or improving documentation, your
efforts help make `pathetic` better for everyone.

## Getting Started

Before contributing, please take a moment to review this guide to ensure a smooth and collaborative process.

### Prerequisites

- **Java**: Ensure you have Java 8 or higher installed.
- **Build Tool**: Use Maven to build the project.
- **Git**: Familiarity with Git for cloning the repository and submitting pull requests.

Clone the repository to get started:

```bash
git clone https://github.com/bsommerfeld/pathetic.git
cd pathetic
```

## Contribution Guidelines

We value quality and collaboration. Please follow these guidelines to ensure your contributions align with the goals of
`pathetic`:

- **Code Formatting**: Maintain the project's existing code style. Run the build process to verify formatting before
  submitting your changes.
- **One Feature per Pull Request**: Focus each pull request (PR) on a single feature, bug fix, or improvement to
  streamline reviews.
- **Cross-Platform Support**: Ensure your changes are compatible across all supported platforms (e.g., Windows, macOS,
  Linux). Test thoroughly or document any platform-specific limitations.
- **Tests**: Include unit tests for new features or update existing tests when modifying functionality to maintain
  reliability and coverage.
- **Discuss Larger Changes**: For significant features or architectural changes, open
  a [GitHub Issue](https://github.com/bsommerfeld/pathetic/issues) to discuss your plan with the maintainers before
  starting work.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the `pathetic` repository.
2. **Create a Branch**: Use a descriptive branch name, e.g., `feature/add-diagonal-movement` or
   `fix/bug-in-node-validation`.
3. **Make Changes**: Implement your feature, bug fix, or documentation update, following the guidelines above.
4. **Test Your Changes**: Run the test suite (`mvn test`) to ensure everything works as expected.
5. **Commit Your Changes**: Write clear, [concise commit messages](https://www.conventionalcommits.org/en/v1.0.0/) (
   e.g.,
   `feat: Add support for diagonal movement in A* algorithm`).
6. **Submit a Pull Request**:
    - Push your branch to your fork.
    - Open a PR against the `trunk` branch of `bsommerfeld/pathetic`.
    - Include a clear description of your changes, referencing any related issues (e.g., `Closes #123`).
7. **Respond to Feedback**: Address any review comments promptly to keep the process moving.

## Code of Conduct

Be kind and respectful in all interactions, whether in issues, PRs, or other discussions. We aim to foster a welcoming
and inclusive community.

See more here: https://github.com/bsommerfeld/pathetic/blob/trunk/CODE_OF_CONDUCT.md
